# Pulse Music Provider Options

## Evaluation Rule

Pulse should only integrate vendors that provide explicit commercial-use rights, edit/remix permissions suitable for short-form social content, proof of license, and clear terms for user-generated content distribution.

## Providers

### Soundstripe

- Fit: Strong candidate for creator/business music licensing.
- Integration posture: Treat as partner/API or licensed catalog metadata only until legal confirms API access and usage rights.
- Pulse action: pursue business/API conversation and require written permission for in-app user selection.
- Source: https://www.soundstripe.com/

### Epidemic Sound Partner/Enterprise

- Fit: Strong candidate if enterprise/partner terms cover platform users and social distribution.
- Integration posture: Do not assume normal creator subscription rights cover Pulse users. Need enterprise agreement.
- Pulse action: pursue partner/enterprise review before importing metadata.
- Source: https://www.epidemicsound.com/

### Feed.fm / Feed Clips

- Fit: Strong candidate for licensed music experiences and short clips.
- Integration posture: promising for productized licensed music APIs, but use only under contract.
- Pulse action: evaluate Feed.fm API/licensing terms for short-form creator content.
- Source: https://feed.fm/

### Sounds HQ

- Fit: Unknown until vendor rights and API terms are confirmed.
- Integration posture: block by default unless documentation/contract proves commercial and edit rights.
- Pulse action: request official API/licensing docs before integration.

### Jamendo Licensing

- Fit: Candidate for licensed commercial music catalog.
- Integration posture: use licensing products only, not random public downloads.
- Pulse action: evaluate catalog metadata import and license proof workflow.
- Sources: https://licensing.jamendo.com/ and https://developer.jamendo.com/

## Recommendation

Start with Pulse originals and metadata-only approved catalog imports. Add one provider at a time after legal approval, beginning with vendors that can provide explicit platform rights, API access, reporting requirements, and proof artifacts.
